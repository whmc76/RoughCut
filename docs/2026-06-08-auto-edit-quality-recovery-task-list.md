# 自动剪辑质量恢复开发任务清单

日期：2026-06-08

## 使用方式

- 本清单按依赖顺序排列，默认先做 `P0` 和 `P1`。
- 目标是尽快恢复 `90` 分左右的批量自动剪辑生产力，不以一次性做满所有长尾优化为目标。
- 每个任务都要有真实任务验证，不接受只在单测里“看起来通过”。

## 里程碑

| 里程碑 | 目标 | 预期结果 |
|---|---|---|
| `P0` | 止血与恢复可用性 | 手动编辑器可用，行为可追踪 |
| `P1` | 文本/切分/投影归位 | 字幕展示稳定，可解释 |
| `P2` | 候选型自动剪辑恢复 | 规则行为一致，误删显著下降 |
| `P3` | 智能删减质量提升 | 废片句段识别和多模态复核上线 |
| `P4` | 评测与发布门禁 | 可持续迭代和质量回归防线 |

## P0 止血与恢复可用性

### T0.1 手动编辑器 `500` 根因修复

- 优先级：`P0`
- 规模：`M`
- 依赖：无
- 交付：
  - 找到 `/manual-editor` 与 `/manual-editor/readiness` 的第一坏层。
  - 修复真实任务的 session/readiness 构造失败。
  - 保证真实任务可以稳定打开编辑器。
- 验收：
  - 至少 `3` 个真实 job 成功加载 manual editor。
  - 不再出现 `500`。

### T0.2 去掉跨阶段 fallback 串写

- 优先级：`P0`
- 规模：`M`
- 依赖：`T0.1`
- 交付：
- 明确哪些 fallback 还会改 projection、source subtitle basis、edit inputs。
- 删除或收紧这些 fallback。
- 为允许存在的 fallback 增加显式诊断字段。
- 已完成：`manual-editor/apply` 不再将 `validate_projected_subtitles_against_source(...).subtitles` 覆盖回写到可编辑轨迹；仅输出 mismatch/fallback 诊断。
- 已完成：`pipeline/steps.py` 中统一验证入口 `_validated_subtitle_projection_for_timeline` 增加 `apply_repair` 开关，默认不改写输入，只有明确 `apply_repair=True` 的流程才使用修复后字幕；平台打包路径默认不改写，避免手工覆盖字幕链路被自动校验悄悄重算。
- 验收：
  - 任意任务都能回答“当前投影来自哪里”“是否触发 fallback”“fallback 改了什么”。

### T0.3 候选/投影/删除来源诊断面板

- 优先级：`P0`
- 规模：`S`
- 依赖：`T0.2`
- 交付：
  - 候选记录 `stage`、`rule_id`、`match_surface`、`risk_level`。
  - 前端可查看高亮/删除的来源说明。
- 验收：
  - 对任意棕色、灰色、绿色高亮，都能定位其来源规则。

### T0.4 建立真实任务 golden set

- 优先级：`P0`
- 规模：`S`
- 依赖：无
- 交付：
  - 选 `5-10` 条真实任务作为回归集。✅ 当前版本已提交 `9` 条锚点。
  - 覆盖开箱、参数介绍、口播密集、展示沉默、产品型号密集等场景。
- 验收：
  - 后续每次重构都可重复跑同一组任务并比对关键指标。

## P1 文本、切分、投影归位

### T1.1 锁定三层文本表面

- 优先级：`P1`
- 规模：`M`
- 依赖：`T0.2`
- 交付：
  - 明确 `raw / canonical / display` 三层文本字段和消费者。
  - 清理错误的跨层读取。
- 验收：
  - 规则匹配、内容理解、字幕显示分别只读自己的目标表面。

### T1.2 切分唯一归属到 Subtitle Segmentation

- 优先级：`P1`
- 规模：`L`
- 依赖：`T1.1`
- 交付：
  - 梳理所有会重切字幕的代码路径。
  - 保证切分只发生一次。
  - 手动编辑模式展示自动切分后的投影。
- 验收：
  - 相同 job 在自动模式与手动模式看到同一版切分基线。

### T1.3 收敛 Subtitle Projection 为只读展示层

- 优先级：`P1`
- 规模：`M`
- 依赖：`T1.2`
- 交付：
  - 投影阶段只做换行、展示样式、可读性控制。
  - 不再承担删词、删段、纠术语。
- 验收：
  - 投影改动不会再改变 edit candidate 或 keep/remove 决策。

### T1.4 字幕格式与术语呈现规范

- 优先级：`P1`
- 规模：`M`
- 依赖：`T1.1`
- 交付：
  - 统一英文型号 + 阿拉伯数字规则，例如 `UHD20`、`EDC17`。
  - 建立品牌/型号显示正则与词表格式化层。
- 验收：
  - golden set 中常见型号格式错误显著下降。

## P2 候选型自动剪辑恢复

### T2.1 规则注册表统一化

- 优先级：`P2`
- 规模：`M`
- 依赖：`T1.1`
- 交付：
  - 统一 filler、catchphrase、pause、repeated_speech、smart_delete 的候选接口。
  - 删除各自私有的隐式删除路径。
- 验收：
  - 规则卡片计数、高亮计数、候选计数一致。

### T2.2 语气词拆成独立语气/句头语气/句尾语气

- 优先级：`P2`
- 规模：`M`
- 依赖：`T2.1`
- 交付：
  - 独立语气默认勾选。
  - 句头和句尾语气单独可配。
  - 三者互不串扰，不与口头禅联动。
- 验收：
  - 切换任一复选项只影响对应子规则。

### T2.3 口头禅独立候选链

- 优先级：`P2`
- 规模：`M`
- 依赖：`T2.1`
- 交付：
  - 口头禅使用独立词表和独立候选类别。
  - 增加句法完整性保护，避免误删正常短语。
- 验收：
  - 不再出现未配置词也被标成口头禅的情况。

### T2.4 语义完整性与边界自然度审计

- 优先级：`P2`
- 规模：`L`
- 依赖：`T2.2`, `T2.3`
- 交付：
  - 对候选删除前后做句法/语义残缺检查。
  - 对句尾收音和句头残片做风险加权。
- 验收：
  - 误删正常内容和硬切收音明显下降。

### T2.5 自动应用阈值与人工阻断门禁

- 优先级：`P2`
- 规模：`M`
- 依赖：`T2.4`
- 交付：
  - `low risk` 自动应用。
  - `medium/high risk` 进入候选池或阻断。
  - 智能删减默认更保守。
- 验收：
  - 自动模式误删率下降，人工撤销率下降。

## P3 智能删减质量提升

### T3.1 废片句段识别器

- 优先级：`P3`
- 规模：`L`
- 依赖：`T2.5`
- 交付：
  - 针对“回删片段、重来片段、低信息废话、口播中断”输出候选。
  - 不直接删除，只进入高风险候选池。
- 验收：
  - 在 golden set 上补到当前明显漏删的废片区。

### T3.2 多模态高风险候选复核

- 优先级：`P3`
- 规模：`L`
- 依赖：`T3.1`
- 交付：
  - 仅对高风险候选调用视频语义复核。
  - 输出“保留/可删/需人工确认”。
- 验收：
  - 文本低信息但画面有展示价值的误删显著下降。

### T3.3 创作者反馈回灌

- 优先级：`P3`
- 规模：`M`
- 依赖：`T2.5`
- 交付：
  - 记录人工撤销和确认样本。
  - 让后续阈值和词表能按创作者风格调整。
- 验收：
  - 同类任务上的人工重复纠偏减少。

## P4 评测与发布门禁

### T4.1 全链路评测脚本

- 优先级：`P4`
- 规模：`M`
- 依赖：`T0.4`, `T2.5`
- 交付：
  - 从 job 批量拉取字幕、候选、删除、投影、最终成片信息。
  - 输出统一评测报告。
- 验收：
  - 任意改动后可一键得到质量对比。

### T4.2 自动剪辑评分卡

- 优先级：`P4`
- 规模：`S`
- 依赖：`T4.1`
- 交付：
  - 产出功能可用性、字幕质量、剪辑质量、生产力四维评分。
- 验收：
  - 新版本分数可与当前 `62/100` 基线直接比较。

### T4.3 回归门禁

- 优先级：`P4`
- 规模：`S`
- 依赖：`T4.2`
- 交付：
  - 对 manual editor 可用性、误删率、字幕缺失率设置阻断门槛。
- 验收：
  - 关键指标回退时，不能直接合入主链。

## 推荐执行顺序

1. `T0.1 -> T0.2 -> T0.3`
2. `T1.1 -> T1.2 -> T1.3 -> T1.4`
3. `T2.1 -> T2.2 + T2.3 -> T2.4 -> T2.5`
4. `T0.4 -> T4.1 -> T4.2 -> T4.3`
5. `T3.1 -> T3.2 -> T3.3`

## 本轮建议先做的高回报任务

- `T0.1` 手动编辑器 `500` 根因修复
- `T0.2` 去掉跨阶段 fallback 串写
- `T1.1` 锁定三层文本表面
- `T1.2` 切分唯一归属
- `T2.1` 规则注册表统一化
- `T2.2` 语气词三分法
- `T2.5` 自动应用阈值与人工阻断门禁

这组任务完成后，系统应先恢复“可用且可解释”，再继续追求更聪明的废片识别。
