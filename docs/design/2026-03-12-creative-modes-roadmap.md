# Creative Modes 路线图

日期：2026-03-12  
状态：Draft

## 背景

RoughCut 当前主链路更接近“已有视频的自动粗剪与包装”。这次扩展不应该把三类能力硬塞进一条流水线里，否则后续会出现：

- 长文本转视频和已有视频粗剪共享了不该共享的输入假设。
- 数字人解说和 AI 导演这种通用增强能力无法复用到不同题材。
- TTS、数字人、语音克隆、素材库策略散落在各个步骤里，后面会越来越难扩展。

因此这里拆成两层：

- 基础工作流：定义任务从什么输入出发。
- 通用增强能力：定义可以叠加到成片上的增强模块。

## 模式拆分

### 1. 长文本转视频模式

定位：独立基础工作流，不是当前“上传已有视频后粗剪”的轻量变体。

目标：

- 根据长文本自动拆章节、抽镜头、拼接素材。
- 使用较新素材库，不走老旧素材。
- 接入 TTS 输出长视频配音。
- 可按创作风格路由不同类型素材。

建议输入：

- 正文全文
- 文章标题 / 副标题
- 风格标签
- 目标时长
- 平台目标
- 配音方案

推荐子流程：

1. 文本结构化：章节拆分、情绪段、信息密度评分、节奏标记。
2. 镜头脚本化：每段产出镜头类型、素材关键词、字幕段、旁白段。
3. 素材检索：优先使用新素材；按时间、清晰度、题材相似度、热度筛选。
4. TTS 合成：Edge TTS 先做默认方案，高级档走语音克隆 API。
5. 长视频编排：镜头拼接、转场、BGM、字幕、封面标题。

风格路由：

- 轻松幽默：段子、梗图、夸张反应、搞笑切点，参考半佛仙人式表达。
- 严肃纪实：新闻、纪录、资料画面、信息卡片，参考智能路障式表达。

当前决策：

- 本轮不接入主任务创建。
- 先保留完整方案和模式注册表。
- 等文本脚本输入页、素材检索器、TTS 编排器成熟后再落地。

### 2. 数字人解说模式

定位：通用增强能力，可叠加到任意成片。

目标：

- 在成片中增加画中画数字人讲解位。
- 支持用户设定形象、机位、比例、出镜频率。
- 用于补信息、补转场、补观点、补品牌感。

建议能力边界：

- 输入：形象 ID、口播文本、出镜模板、出现时机、遮挡避让规则。
- 输出：带 alpha 的数字人片段或带背景成品片段。
- API 适配：HeyGem 优先，也保留其他在线数字人 API 抽象层。

推荐子流程：

1. 时间轴分析：找出适合插入解说的位置。
2. 解说脚本生成：可直接沿用原台词，也可让 AI 导演先润色。
3. 数字人渲染：根据形象和机位模板生成口播段。
4. 画中画合成：与字幕、包装、标题避让。

落地优先级：

- P1：任务层支持启用数字人解说模式，并保留配置结构。
- P2：输出解说片段规划和画中画占位信息。
- P3：接 HeyGem 等 API，进入真实合成。

### 3. AI 导演模式

定位：通用增强能力，可叠加到任意成片。

目标：

- 自动校正台词逻辑。
- 适当补充科普信息、情绪桥段、总结桥段。
- 保持说话人音色一致，完成重配音。
- 让视频更像“被导演过”，而不是简单剪掉停顿。

推荐能力结构：

- 台词分析：识别铺垫、爆点、信息缺口、逻辑跳跃。
- 台词改写：加强钩子、过渡、总结、互动收口。
- 语音重建：Edge TTS 做快速版，语音克隆 API 做高质量版。
- 节奏回贴：把改写音频重新回贴到时间线。

方法论要求：

- 知识型视频：优先补事实背景、机制解释、对比结论。
- 情绪型视频：优先补氛围、抒情、人物动机。
- 爆款表达：强化开头钩子、中段反转、结尾观点或互动。

落地优先级：

- P1：任务层支持启用 AI 导演模式，并把模式上下文写入内容画像与渲染计划。
- P2：补“导演建议稿”和“重配音计划”。
- P3：接入 TTS / 语音克隆并落真实重配音。

## 基础技术策略

### TTS 与语音克隆

- 快速默认：Edge TTS
- 高级档：RunningHub API 或其他语音克隆 API
- 策略：统一成 `voice_provider` 抽象，按速度、成本、音色一致性做切换

### 数字人 API

- 首选：HeyGem
- 兼容：其他在线数字人 API
- 策略：统一成 `avatar_provider` 抽象，输入只保留形象、模板、脚本、音频

### 素材库策略

- 必须记录素材时间、题材、清晰度、版权来源
- 检索时默认优先新素材
- 老旧素材不禁用，但不能成为默认召回优先级

## 本轮实施范围

本轮只落这三件事：

1. 创作模式注册表：明确标准成片、长文本转视频、数字人解说、AI 导演。
2. 任务模型接线：任务创建时可选择基础模式和增强模式。
3. 任务上下文注入：把创作模式写入内容画像与渲染计划，后续可直接扩展真实执行器。

明确不做：

- 长文本转视频真实生成
- 数字人真实 API 合成
- AI 导演真实重配音
