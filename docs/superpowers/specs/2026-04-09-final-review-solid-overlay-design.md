# 最终审核实体弹层设计

## 背景

预审核弹层已经切换到暖白实体工作台，但最终审核仍保留旧的卡片层级和容器承载，视觉上与预审核不一致。用户要求将其他审核弹窗统一，因此本次只收敛到 `final_review` 弹层。

## 目标

- 让最终审核弹层与预审核使用同一套暖白实体工作台外壳。
- 将预览区、审核证据、退回说明、质量结果、字幕抽检统一为浅色实体卡片体系。
- 保持最终审核的字段顺序、交互、按钮语义和逻辑不变。

## 方案

### 1. 容器统一

- `JobReviewOverlay` 在 `final_review` 场景下增加和预审核对应的实体面板容器类。
- 外围仍复用当前半透明暗罩，不新增 blur。

### 2. 内部卡片统一

- `JobFinalReviewOverlay` 根节点保持 `panel`，但补充最终审核专用表面类。
- 预览框、审核证据卡、当前确认区、质量结果区、字幕抽检卡统一使用浅色实体卡片。
- 退回说明 textarea 与问题分类按钮落在同一视觉卡片中，避免旧样式混杂。

### 3. 非目标

- 不修改最终审核的通过/退回逻辑。
- 不重排区块顺序。
- 不扩展到创建任务弹窗或非审核弹窗。

## 验收标准

- 打开最终审核时，视觉上与预审核属于同一套弹层系统。
- 预览、证据、退回说明、质量结果、字幕抽检区块都不再混用旧的深色/透明承载。
- 最终审核现有测试继续通过，并新增结构测试锁住实体工作台类名。
