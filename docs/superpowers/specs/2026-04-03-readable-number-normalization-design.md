# 可读型数字转写设计

## 目标

在不新增文本轨道的前提下，统一 RoughCut 在 ASR 后处理和 LLM 字幕校对阶段的数字写法策略：

- `text_raw` 保留原始生成文本
- `text_norm` / `text_final` 复用同一套数字规范化逻辑
- 信息型数字优先阿拉伯数字
- 成词组的自然口语数量表达保留汉字

## 设计原则

1. 不把数字策略继续分散到 ASR 正则、LLM prompt、LLM 后清洗三处各自维护。
2. 以“可读型”为默认倾向：
   - `2个档位`、`3月5号`、`8点20`、`A4纸` 优先标准化
   - `一个`、`一次`、`两三个` 这类自然口语词组保留汉字
3. 不新建额外数据轨道，只复用现有 `text_raw / text_norm / text_final`。

## 规则分层

### 1. 信息型数字

优先转阿拉伯数字：

- 百分比
- 日期
- 时间
- 版本代号 / 序号 / 档位
- 规格单位
- 字母+数字组合

### 2. 自然口语词组

优先保留汉字：

- `一个`
- `一次`
- `一年 / 一月 / 一天 / 一小时 / 一分钟 / 一秒`
- `两三个`、`一两个`、`七八个` 这类模糊数量

### 3. 冲突裁决

当 `个` 后面直接跟信息型名词时，转成阿拉伯数字，例如：

- `一个接口` -> `1个接口`
- `两个档位` -> `2个档位`

当 `个` 只是自然口语成词时，保留汉字，例如：

- `一个整体效果`

## 暂不覆盖

- 复杂价格口语，如 `19块9`
- 长型号跨词识别，如 `g p t 四 o`
- 更复杂的时间口语，如 `差十分钟八点`
