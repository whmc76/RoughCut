# 任务页创建弹窗设计

## 目标

把任务页中的内嵌创建区域移出主页面，改为通过顶部“创建任务”按钮打开悬浮窗口，减少为了创建任务而向下滚动的操作成本。

## 现状

- `frontend/src/pages/JobsPage.tsx` 通过顶部按钮切换 `createOpen`，仅控制页面底部的内嵌创建区展开或收起。
- 创建流程由两个现有模块组成：
  - `ConfigProfileSwitcher`：选择和管理剪辑方案。
  - `JobUploadPanel`：选择素材并提交创建任务。
- 列表和创建区共存于一个长页面里，创建动作需要滚动到下方完成。

## 设计决策

### 交互

- 顶部按钮统一使用“创建任务”文案。
- 默认不在任务页正文中渲染内嵌创建区。
- 点击顶部按钮后打开悬浮窗口。
- 悬浮窗口内按顺序展示：
  - 剪辑方案
  - 创建任务
- 点击遮罩、关闭按钮或按 `Escape` 关闭窗口。
- 创建成功后自动关闭窗口，并保持现有“选中新建任务”的行为。

### 结构

- 新增专用创建弹窗组件，避免把 `JobsPage` 的展示逻辑和弹窗壳体耦合在一起。
- 复用现有 `ConfigProfileSwitcher` 与 `JobUploadPanel`，不重写表单内容。
- 任务页只保留任务列表、活动任务和审核/详情弹窗。

### 样式

- 复用现有 floating modal 视觉体系，保持和审核弹窗一致的悬浮层体验。
- 在弹窗内容中使用双区块布局，桌面端左右分栏，窄屏下改为上下堆叠。

## 验收标准

- 任务页初始渲染时看不到内嵌创建区域。
- 点击顶部“创建任务”按钮后可看到包含“剪辑方案”和“创建任务”的弹窗。
- 关闭弹窗后相关模块从页面移除。
- 创建成功后弹窗自动关闭。
